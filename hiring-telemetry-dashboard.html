Hiring Telemetry Dashboard

A hiring telemetry dashboard is not a report. It is a live operational view — showing the health of every active mandate simultaneously, updating in real time as new signals arrive.

What a Hiring Telemetry Dashboard Shows

The Majhi OS hiring telemetry dashboard is the primary interface for real-time mandate health monitoring. Unlike ATS dashboards that display historical candidate data, the telemetry dashboard displays live operational signals — updating continuously as outreach interactions, pipeline transitions, and recruiter activity generate new data.

Six Views in the Majhi OS Telemetry Dashboard

🏥

Mandate Health Panel

A live grid showing the current health score for every active mandate — color-coded green (above 65), amber (50–64), or red (below 50). The primary instrument panel for TA leaders and search directors.

📈

Response Decay Monitor

Time-series view of outreach response rates across all active mandates — showing current rate, decay trajectory, and projected rate at current decay speed. Highlights mandates approaching the danger threshold.

⚡

Funnel Velocity Tracker

Stage-by-stage pipeline velocity for each mandate — showing actual velocity vs benchmark for each stage and flagging stages running more than 20% slower than benchmark.

👥

Recruiter Load Map

Visual representation of each recruiter's current load index — showing capacity utilization, mandate distribution, and overload risk. Allows immediate rebalancing decisions.

⌛

SLA Breach Tracker

Time-to-breach estimates for every mandate's close-date SLA — sorted by breach probability and time remaining. The escalation trigger for mandates approaching breach threshold.

🔄

Recovery Status Queue

Live status of all active recovery sequences — showing which interventions are in progress, how long they have been active, and whether the health trajectory is improving as expected.

The Design Principle: Command Center, Not Spreadsheet

The telemetry dashboard is designed to feel like mission control — not a CRM or ATS report view. The design principle: a TA director should be able to look at the dashboard for 60 seconds and know exactly which mandates need attention, what the specific attention they need is, and what the recommended action is. Every element is designed to support that 60-second decision cycle.

"The best test of a telemetry dashboard is what happens when you look at it at 9am Monday morning. In 60 seconds, you should know the exact operational state of every search you are responsible for. If you cannot do that in 60 seconds, it is not a telemetry dashboard — it is a reporting tool." — Manas Majhi

Who Uses the Dashboard and How

TA leaders and search directors use the dashboard as their primary operational interface — checking it multiple times daily, reviewing alerts as they surface, and authorizing recovery actions from within the dashboard. CHROs and VP People use it for weekly strategic reviews — scanning health scores and SLA proximity across all mandates. CEOs use the executive summary view for monthly hiring health reviews.

Frequently Asked Questions

What does a hiring telemetry dashboard show?

A hiring telemetry dashboard shows the live operational state of every active mandate — health scores, response decay trajectories, funnel velocity by stage, recruiter load distribution, SLA proximity, and recovery sequence status. All in real time, not in weekly reports.

How is the Majhi OS dashboard different from an ATS dashboard?

An ATS dashboard shows candidate stages and historical pipeline data. The Majhi OS telemetry dashboard shows live operational signals — what is happening right now across all active searches — and surfaces alerts and recommended actions when signals cross thresholds.

How often does the dashboard update?

Continuously. Health scores recalculate whenever any input metric changes. Response decay and funnel velocity update with every outreach interaction and pipeline event. The dashboard reflects the current state of the hiring system, not the state from the last batch-processing cycle.

Can I see all mandates in one view?

Yes. The Mandate Health Panel shows the current health score for every active mandate in a single grid view — color-coded by health status. From this view, TA directors can drill into any specific mandate to see its individual telemetry signals and active alerts.

Does the dashboard include actionability or just data?

Both. Each alert in the dashboard comes with a specific recommended action — not just 'Mandate 3 is at risk' but 'Mandate 3 response decay has crossed threshold; recommended action: activate Outreach Recovery Sequence B (74% probability of recovery).' The dashboard is designed for decision-making, not passive monitoring.
